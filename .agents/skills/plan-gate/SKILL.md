# plan-gate: 最弱読者に復唱させ、差分で曖昧さを名指しする

計画の明確さは、書いた本人には判定できない（本人には全部自明に見える）。
実行者と同格以下の「最弱読者」に読ませ、復唱と意図の差分を**計画の欠陥リスト**として使う。

## 原理（なぜ「明確か?」と聞かないか）

- 弱いモデルに判定させると同意バイアスで通る。判定者ではなく**読者**として使う
- **復唱はズレるのが前提**。ズレは失敗ではなく検出信号であり、
  モデルの能力不足ではなく計画の曖昧さとして扱う。直すのは常に計画の側

## 手順

1. **適用判断**: 計画は実行境界（下位モデル委譲 / handoff / 別AIツール / 人間実行）を越えるか?
   越えないなら掛けない（立てた本人が即実行する計画には純オーバーヘッド）。
   3ステップ未満の自明な計画にも掛けない
2. **復唱依頼（最下層モデルへ・1回だけ）**: 計画本文のみを渡し、次の3点だけ出力させる。
   「この計画は明確ですか」とは聞かない:
   - (a) 各ステップを自分の言葉で1行ずつ言い直す
   - (b) 最初に実行する具体的操作を1つ挙げる
   - (c) 意味が取れない語・複数の解釈ができる箇所を列挙する
3. **差分照合（指揮者）**: 復唱と自分の意図を突き合わせ、ズレた箇所を**計画の側で**修正する。
   修正手段は「用語の定義追加・手順の分割・前提の明文化」。計画の狙い自体は変えない
4. **終了**: 再復唱はしない（呼び出し上限1回）。修正済みの計画をそのまま実行境界へ渡す

## トークンガード

- 最下層モデルの呼び出しは計画1本につき1回。往復対話にしない
- 渡すのは計画本文のみ。コードベース・経緯・会話履歴を添付しない
- ズレが検出されなくても再確認のための追加呼び出しをしない（それは判定依頼への逆戻り）
