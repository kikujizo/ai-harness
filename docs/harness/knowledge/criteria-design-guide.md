# 基準設計ガイド — 効く基準ファイルの作り方（rubric engineering）

基準ファイルは再帰照合ループの「外部信号」であり、**基準の質がループの収束品質の天井**になる。
LLM-as-judge 研究の実証知見を、criteria/ を書くときの規律に落とす。

## 1. 5つの設計規律（研究的裏付けつき）

1. **バイナリで書く（○か×か）**
   3段階以上の部分点は判定のブレを生むだけで識別力は上がらない。実証では2値化により判定者間の一致率が約20ポイント向上する。
   ×「読みやすさ: 5段階」→ ○「1段落に主張が2つ以上入っている段落が存在する: ○/×」
2. **観測手順まで書く（形容詞を禁止する）**
   「役に立つ」「高品質」は判定不能。「どこを見て・何があれば○か」の手順に開く。
   ×「受け入れ条件が明確」→ ○「各受け入れ条件が、実行して見える事象（コマンド結果・画面表示・ファイル存在）で書かれている」
3. **長さ中立を明記する**
   LLM判定者には冗長な出力を高く評価するバイアスが実測されている。基準ファイルに1行入れる:「簡潔な成果物は冗長な成果物と同等以上に評価する」。
4. **判定は4部構成で依頼する**
   基準の定義 → 1項目ずつの照合を強制する手順 → ○×の判定規則 → 判定不能時の扱い。
   recursive-review スキルの「基準復唱 → 1項目ずつ照合 → 名指し」はこの構造の実装。
5. **人間で較正する（キャリブレーション）**
   新しい基準ファイルは、最初の2〜3回は人間がAIの○×判定を確認する。人間とAIの判定が割れた項目は基準の書き方が悪い（曖昧）ので、その項目を書き直す。
   割れなくなったら自動運用に昇格（自律度 L1→L2 の昇格と同じラチェット）。

## 2. 帰属区分（新規昇格分のみ）

P3（Issue #73）マージ後に新規作成・新規昇格する基準項目には、
[reflux.md](reflux.md) §3層帰属区分に従い `attribution` と `source` を記録する。

- `source-derived`: 外部一次資料由来。原典URLまたは識別可能な出所を必須。
- `operationalization`: 操作的解釈。根拠となる原典または上位知見と ai-harness 側の追加解釈を分けて記録。
- `repository-policy`: リポジトリ固有運用。採用理由と適用範囲を必須（外部URLは不要）。

出所不明の知見は criteria へ昇格させない。既存 `docs/criteria/` への遡及適用はしない。
記入様式は [templates.md](../../templates.md) の基準ファイルテンプレートを正本とする。

## 3. 基準ファイルのライフサイクル

```
誕生: docs/ で同じ知見を2回参照した → 基準ファイルとして起工する（[reflux.md](reflux.md) のゲート①）
較正: 2〜3回は人間がAI判定を確認 → 割れた項目を書き直す
運用: recursive-review / recursive-writing の外部信号として自動使用。
      レビューで「基準外の気づき」が3回同内容なら、既存項目に追記する（[reflux.md](reflux.md) のゲート②）
剪定: 6ヶ月間一度も×を出していない項目は削除候補（全員が守れている＝もう「目」でなくてよい。
      ただし削除は Decision Log に残す——守れている理由が基準の存在自体かもしれないため）
昇格: 基準がツール・プロジェクト横断で普遍化したら AGENTS.md へ（[reflux.md](reflux.md) のゲート③）
```

昇格・剪定の数値ゲートは [reflux.md](reflux.md) が正本。

## 4. アンチパターン

| パターン | 症状 | 直し方 |
|---|---|---|
| 形容詞の羅列 | 全項目○になり照合が儀式化する | 観測手順に開き直す（規律2） |
| 項目インフレ | 10項目超で照合コストが品質向上を上回る | 6ヶ月×なしの項目を剪定。1ファイル10項目以内 |
| 部分点の導入 | 「7割満たしている」等の交渉が始まる | バイナリに戻す（規律1） |
| 基準の自己増殖 | AIが照合のたびに新項目を勝手に追加する | 追加はゲート②（「基準外の気づき」3回同内容）+ 人間承認のみ（メタループ L1固定） |
| 借り物の基準 | 外部の立派なチェックリストを丸ごと導入して使われない | 自分のレビューで実際に×が出た項目だけ残す |

## 5. 基準は「読み物」ではなく「照合の台本」

参照させるだけでは効かない。**書かせて → 点検させて → 直させるループ**で初めて効く。
LLM-as-judge 研究の言葉では「基準は判定手順として運用されて初めて出力に効く」と言い換えられる。
基準ファイルは読み物ではなく、**照合という行為の台本**として書くこと。
