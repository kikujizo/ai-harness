# トレンドウォッチ — 外部変化を成果物の鮮度に変換するループ

`../knowledge/reflux.md` が「自分の成果物の鮮度」（SSOT Rot対策）を守るのに対し、
このループは「外部環境の鮮度」を守る。利用しているAIツールの仕様・課金・機能は数週間単位で変わり、
**ハーネスの前提が音もなく崩れる**（例: 課金体系の変更はコスト方針を直撃する）。

運用イメージ: 利用しているAIツール群の仕様変動を月次で棚卸しし、記録として残す
（例: `YYYY-MM_snapshot.md` のような月次ファイルを積み上げていく。古いものは消さない。変化の履歴が価値になる）。

## ループの定義

- **トリガー**: 月1回、またはツールの大型発表を見かけたとき（人間起動）
- **手順**: 利用しているAIツール × 3観点（課金・機能・制限）で確認
  → 「実害basis」で選別（下記基準）→ 記録化 → 影響する成果物を名指しで更新
- **選別基準（これを満たさない話題は捨てる）**:
  1. ハーネスの前提（コスト方針・役割分担・ルーティング）を変えるか
  2. 現在のギャップ（無進展検出・計測）を安く埋める新機能か
  3. 放置すると事故る変更（廃止・仕様変更・制限強化）か
- **上限**: 網羅ではなく選別（ニュースまとめを作らない）。1回の記録は選別を通ったものだけ
- **自律度**: L1固定（AIは調査と起案まで。ハーネス本体の書き換えは人間承認必須）
- **無進展検出**: 2回連続で「対応アクションゼロ」なら頻度を落とす（例: 月次→隔月）

## 台帳エントリ例

台帳の正式書式は [../loops/ledger-template.md](../loops/ledger-template.md) を正本とする。
以下はこのループを台帳に登録する場合の記入例（イメージ）:

```
ループ名: トレンドウォッチ
トリガー: 月1回、またはツールの大型発表を見かけたとき（人間起動）
手順: 利用AIツール×3観点(課金/機能/制限)を確認 → 選別基準3条件で判定 → 記録 → 影響成果物を名指し更新
自律度: L1（調査・起案まで。ハーネス本体の書き換えは人間承認）
無進展検出: 2回連続アクションゼロで頻度を落とす
```
