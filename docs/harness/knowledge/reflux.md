# 知識還流ループ — 使うほど賢くなる外部信号

目的関数: **「同じ知見を二度と会話で説明しない」**。
知見は使用実績に基づいて段階的に昇格し、使われないものは滞留させず剪定する。
このループの自律度は**恒久L1**（AIは昇格候補の提案までで、常時適用ルールの書き換えは人間が承認する）。

セッション終了時・レビュー完了時にこのループを回す手順は `.agents/skills/knowledge-reflux/SKILL.md`。

## 昇格パイプライン

知見は下の4段階を**1段階ずつ**上がる。飛び級（生メモから直接AGENTS.mdへ）は禁止。

```
[生メモ]     任意のローカルノート・その場のメモ（構造化されていない着想）
   ↓ 随時: 人間が「使える」と判断したとき
[docs/]      リポジトリの docs/ に構造化・出典つきで記録（読み物として参照可能）
   ↓ ゲート①: 同じ知見を2回参照した → 新しい基準ファイル化
[criteria/]  照合可能な基準ファイルに変換（バイナリ + 観測手順。docs/criteria/）
   ↺ ゲート②: レビューで「基準外の気づき」が3回同内容 → 既存基準ファイルへ項目追記
   ↓ ゲート③: その基準がプロジェクト・ツール横断で3回以上×を検出した実績
[AGENTS.md]  常時適用ルールへ内面化（最上位）
   ↓
[剪定]       6ヶ月×なしの基準 / 1年参照なしの知見は削除候補（人間判断・Decision Log必須）
```

## 昇格ゲート（3本立て・数値で固定 — 場当たり的判断の排除）

| ゲート | 遷移 | トリガー（数値ゲート） | 却下条件 |
|---|---|---|---|
| （随時） | 生メモ → docs/ | 人間が「使える」と判断したとき随時 | 元記事の要約で終わるものは不可（自分の運用への含意が1つ以上あること） |
| ① | docs/ → criteria/（新規基準ファイル化） | **同じ知見を2回参照した** | バイナリ + 観測手順に開けない知見は昇格不可（読み物のまま置く） |
| ② | criteria/ 内（既存基準ファイルへの項目追記） | **レビューで「基準外の気づき」が3回同内容** | 基準の自己増殖防止のため追記は人間承認必須。1ファイル10項目以内を超える追記は先に剪定を検討する |
| ③ | criteria/ → AGENTS.md | **横断的に3回以上×を検出した実績** | プロジェクト固有の基準は昇格不可（そのリポジトリの `docs/criteria/` 止まり） |
| 剪定 | ─ | 6ヶ月×なし / 1年参照なし | 削除はDecision Log必須（守れている理由が基準の存在自体である可能性を残すため） |

基準ファイルの作り方は [criteria-design-guide.md](criteria-design-guide.md) に従う。
`docs/criteria/README.md`・`criteria-design-guide.md`・`recursive-review` Skillにある「3回」の記述は、
いずれもこのゲート②（既存基準ファイルへの項目追記）を指す同一の基準である。

## ループ健全性のガード

- **無進展検出**: 3ヶ月間どの遷移も起きていなければ、ループ自体が死んでいる（収集だけのコレクター化）。プロセスレビューで棚卸しする。
- **時間上限**: 生メモの精製は1ノート30分以内。超えるなら知見が生煮え（生メモに留める）。
- **予算**: 精製・変換は逼迫していない枠で行う。
- **分業**: 精製・変換はAI、昇格判断は人間（実績データを見て）。自己内省だけでは昇格しない。
- **停止時の遷移先**: 昇格判断に迷ったら人間へ返す（このループの承認者は常に人間）。

## 経緯レス正本ルール（還流時に必ず適用）

昇格して上位に反映したら、**置き換えた旧記述はその場で削除**する（アーカイブへ移す）。
上位ファイル（criteria・AGENTS.md）に「旧版・移行済み・参照しない」の注記や旧記述を残さない。
経緯の記録は Decision Log が担い、正本にはAIが実運用で読む「今の正解」だけを置く。詳細は [ssot.md](ssot.md) §経緯レス正本ルール。

## なぜこのループが要になるか

自律ループの品質はアンカーファイル（AGENTS.md・prompts・Issue本文の書き方）の品質が天井を決める（[../loops/principles.md](../loops/principles.md) §4）。
そのアンカーを継続的に良くする唯一の供給源がこの知識還流ループである。
開発ループが「今日の成果」を作り、知識還流ループが「明日のループの精度」を作る。
