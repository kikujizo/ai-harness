# recursive-writing: 基準を内面化してから書く

品質が重要な文章タスクで使う。使い捨てのメモや途中経過には使わない（トークンの無駄）。

## 手順

1. **基準の取得**: `docs/criteria/` に対象に合う基準ファイルがあれば読む。なければ、その文章の目的に合う基準を10項目以内で暫定的に書き出し、
   確認を待たずにこの回の照合にすぐ使ってよい（暫定基準）。
   複数回の使用で有用だと分かった場合、またはユーザーが明示的に求めた場合に限り、
   ユーザーに確認してから `docs/criteria/` へ正式な基準ファイルとして昇格保存する（次回から再利用）。
2. **初稿**: 基準を読んだ直後に書く。
3. **照合**: 基準を1項目ずつ初稿に当て、逸脱を「基準番号+該当箇所の引用」で名指しする。
   「概ね問題なし」で済ませない。逸脱ゼロならゼロと宣言する。
4. **修正**: 名指し箇所だけ直した版を出す。全体を書き直さない。
5. **上限と無進展検出**: ループは最大2周。ただし**前の周と逸脱リストが同一なら周回を待たず即停止**
   （このループでは直せない構造的問題のシグナル）。残る逸脱は「残逸脱」として明示し、人間の判断に渡す。

## 注意

- 基準を「参照しました」と言うだけでは効かない。照合(手順3)を省略しない。
  参照と照合の間に差があることがこのスキルの存在理由。
- 修正版だけを最終出力とし、初稿と照合ログは折りたたむか簡潔にする。
