# ループ台帳テンプレート

すべてのループを1枚の台帳（例: リポジトリの `docs/loop-ledger.md`）で管理する。
**台帳にないループは存在してはならない**（野良ループ＝暴走リスク）。
台帳自体の棚卸しは四半期に1回、または新規ループ追加時に行う。

新しいループを設計する手順は `.agents/skills/loop-design/SKILL.md`、
各項目の背景原則は `docs/harness/loops/principles.md` を参照する。

## 記入する7項目

| 項目 | 書くこと |
|---|---|
| トリガー | 何がこのループを起動するか（イベント・スケジュール・人間の起動） |
| 状態 | 現在位置がどこに外部化されているか（ラベル・ファイル。メモリ内だけは禁止） |
| 上限 | 反復・時間・予算の3つ、数値で（`docs/harness/loops/principles.md` §2 の4ハードガード） |
| 停止条件 | 正常終了は何か／異常終了時に誰へ・何が遷移するか |
| 自律度 | L1 / L2 / L3 のどれか、および昇格・降格の条件 |
| 計測 | 何を・どこに記録するか（ゼロタッチ率・フォールバック発動率など） |
| ギャップ | 未実装の防御。なければ「なし」 |

## 台帳エントリの書式

```markdown
## L-{番号}: {ループ名}
- トリガー:
- 状態:（どこに外部化されているか）
- 上限:（反復 / 時間 / 予算）
- 停止条件:（正常終了 / 異常終了 → 遷移先）
- 自律度: L{1|2|3}（昇格・降格の条件）
- 計測:（何を・どこに記録するか）
- ギャップ:（未実装の防御。なければ「なし」）
```

## 記入例（1件）

```markdown
## L-0: メイン開発ループ（Issue → マージ）※記入例
- トリガー: 人間のIssue作成（テンプレがPMレビュー待ちラベルを付与）
- 状態: ラベル状態機械 + Issue/PR本文（SSOT）
- 上限: 修正ラウンド2回、フォールバック各1回、各ステップにtimeout（PM系15分・実装系60分）
- 停止条件: 自動マージ（正常終了） / 自動停止ラベル → 人間へエスカレーション（異常終了）
- 自律度: L2〜L3（通常リスク=L3、高リスク=人間承認）。昇格根拠は直近10回誤動作ゼロ、降格は不可逆カテゴリ接触で即時
- 計測: ゼロタッチ率・マージレイテンシ（マージ要約に機械追記）
- ギャップ: なし
```

## 登録済みループ（本リポジトリ）

## L-1: plan-gate（計画可読性ゲート・最弱読者テスト）

- トリガー: 人間または指揮者の明示指定（lab Skill `.agents/skills/plan-gate/SKILL.md`。自動起動なし）
- 状態: 修正は計画本文（ファイル・Issue本文・handoff資料）へ直接反映。ゲート自体はセッション内で完結。
  実績（検出ズレ数・採用修正数）は委譲先Issue/PRコメントまたはhandoff資料に記録
- 上限: 反復=1ラウンド＋条件付き追い復唱1回 / 時間=読者1呼び出し5分・全体15分 /
  予算=最下層モデル最大3呼び出し（計画本文のみ添付）
- 停止条件: 正常=修正済み計画を実行境界へ渡す / 異常=時間超過は復唱なしで指揮者判断へ切替、
  追い復唱でも過半ズレは計画を分割して出直す
- 自律度: L1（読者は報告のみ。採否・受け渡し判断は指揮者）。昇格はlab共通規則（実案件2回以上・
  重大事故0件でcore昇格候補、Decision Log記録）、重大事故1件で停止
- 計測: 発動ごとの検出ズレ数・採用修正数（委譲先Issue/PRコメント）
- ギャップ: トークン量の自動計測なし（呼び出し回数上限で代替）
