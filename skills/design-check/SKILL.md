# design-check: 実装前に5行だけ立ち止まる

スーパーバイザとして、実装に入る前（または実装をCursorへ振る前）に使う。
重い設計ドキュメントは作らない。判断に必要な最小限だけ書く。

## 5行チェック（必ずこの5行で出す）

1. 方針: {採用するアプローチを1文}
2. 却下案: {検討して選ばなかった案と理由を1文。なければ「なし」}
3. 影響範囲: {変更ファイル見込みと、壊れうる既存機能}
4. リスク: {不可逆4カテゴリ該当の有無 + 最大の懸念1つ}
5. 分割判断: {Issue粒度(3〜5ファイル/半日/条件5項目)に収まるか。超えるなら分割案}

## 分岐

- 却下案があり影響が大きい → `docs/templates.md` のDecision Log草案を追加で出す（採否は人間）
- 自分で実装する → そのまま実装へ。完了時は出力契約で報告
- Cursorへ振る → `docs/templates.md` の「実装指示書」を出力する。
  指示書には「変更してはいけない範囲」と「仕様判断が必要なら実装せずレビュー依頼に含める」を必ず含める

## トークンガード

- 明確で小さいタスク（1〜2ファイル・雑務・文章）にはこのスキルを使わず直接作業する
- 5行チェックを超えて長文化しない。長くなる= Decision Logか仕様差し戻しのサイン
