# plan-gate: 最弱読者に復唱させ、差分で曖昧さを名指しする

計画の明確さは、書いた本人には判定できない（本人には全部自明に見える）。
実行者と同格以下の「最弱読者」に読ませ、復唱と意図の差分を**計画の欠陥リスト**として使う。

## 原理

- 弱いモデルに「明確か?」と**判定させない**（同意バイアスで通る）。判定者ではなく**読者**として使う
- **復唱はズレるのが前提**。ズレは失敗ではなく検出信号であり、直すのは常に計画の側
- **読者は1体では足りない**。1体のズレは「読者のノイズ」と「計画の曖昧さ」を区別できない。
  独立した2体が同じ箇所で転ぶなら、それはモデルの能力不足ではなく計画の欠陥（self-consistencyの応用）
- **言い直しだけでは理解を装える**（語順の入れ替えで復唱風の出力が作れる）。
  「そのステップが終わると何がどうなっているか」の**結果予測**は理解がないと書けない
- コスト構造: 高いのは**ラウンド数**（指揮者＝上位モデルの文脈が燃える）であって、
  最下層モデルの呼び出し数ではない（桁で安い）。**増やしてよいのは並列の読者数、
  増やしてはいけないのはラウンド数**

## 手順

1. **適用判断**: 計画は実行境界（下位モデル委譲 / handoff / 別AIツール / 人間実行）を越えるか?
   越えないなら掛けない（立てた本人が即実行する計画には純オーバーヘッド）。
   3ステップ未満の自明な計画にも掛けない
2. **並列復唱（最下層モデル2体へ同時・1ラウンドのみ）**: 計画本文のみを渡し、
   ステップ番号付きで次の3点だけ出力させる。「この計画は明確ですか」とは聞かない:
   - (a) 各ステップを自分の言葉で1行で言い直し、**そのステップ終了時に何がどうなっているか**を1行で予測する
   - (b) 最初に実行する具体的操作を1つ挙げる
   - (c) 意味が取れない語・複数の解釈ができる箇所を列挙する
3. **差分照合（指揮者）**: ズレを2体の一致で仕分ける:
   - **2体が同じ箇所でズレた** → 計画の欠陥（確定）。計画を修正する
   - **1体だけズレた** → 読者ノイズの疑い。指揮者が該当箇所だけ読み、曖昧なら修正・明確なら流す
   - 修正手段は「用語の定義追加・手順の分割・前提の明文化」。計画の狙い自体は変えない
4. **条件付き追い復唱（最大1回）**: 既定では再復唱しない。次のいずれかのときだけ、
   **修正したステップのみ**を新しい1体に読ませてよい:
   - ズレがステップ数の**過半**に及んだ（骨格レベルの曖昧さ）
   - 計画が**不可逆4カテゴリ**に触れる（定義はルートの`AGENTS.md`）
   追い復唱でも過半がズレるなら、それは書き方でなく計画の粒度の問題。計画を分割して出直す（ループしない）
5. **終了**: 修正済みの計画をそのまま実行境界へ渡す

## トークンガード

- 総予算: **並列2呼び出し＋条件付き追い復唱1回まで**。往復対話にしない
- 渡すのは計画本文のみ（追い復唱は修正ステップのみ）。コードベース・経緯・会話履歴を添付しない
- ズレが検出されなくても確認のための追加呼び出しをしない（それは判定依頼への逆戻り）
