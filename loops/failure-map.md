# 失敗様式マップ — MAST × このハーネスの防御

UC Berkeley の [MAST（Multi-Agent System Failure Taxonomy）](https://arxiv.org/html/2503.13657v2) は、
7つのマルチエージェントフレームワークの1,600超の実行トレースを分析し、失敗を **14様式・3カテゴリ** に分類した
（アノテーター間一致 Cohen's κ=0.88）。経験則ではなく実測データに基づく失敗の地図であり、
このハーネスの防御がどこまで妥当かをこの地図で照合する。

各行の「このハーネスでの防御手段」は、対応する正本ルールへの参照（キット内部のファイルは相対リンク、
導入先の正本はコードスパンのパス）で示す。

## 1. 仕様・システム設計の失敗（41.8% — 最大のカテゴリ）

| MAST失敗様式 | このハーネスでの防御手段（該当ルールへのリンク） | 状態 |
|---|---|---|
| タスク仕様への不服従 (11.8%) | Issue自己完結原則 + 受け入れ条件 + PM評価の差し戻し（ルートの`AGENTS.md` 実装ルール・エスカレーション基準） | 防御あり |
| ステップ反復・無進展 (15.7%) | 反復上限 + **無進展検出**（[principles.md](principles.md) §2 ハードガード3。「前反復と同一結果なら即停止」） | 防御あり |
| 終了条件の不認識 (12.4%) | verdict契約で終了を機械判定（ルートの`AGENTS.md` 出力契約の `PM_VERDICT:` / `REVIEW_VERDICT:`） | 防御あり |
| 役割仕様への不服従 (1.5%) | 役割表 + 「やらないこと」列（ルートの`AGENTS.md` 役割分担） | 防御あり |
| 会話履歴の喪失 (2.8%) | 状態外部化・SSOT（[principles.md](principles.md) §1 要素⑥ / [../knowledge/ssot.md](../knowledge/ssot.md)） | 防御あり |

**含意**: 最大カテゴリ（41.8%）が「モデルの能力」ではなく**組織設計の問題**であること自体が、
「ハーネスとループの設計に投資する」という方針の実証的裏付けになっている。

## 2. エージェント間の不整合（36.9%）

| MAST失敗様式（代表） | このハーネスでの防御手段（該当ルールへのリンク） | 状態 |
|---|---|---|
| 情報の隠蔽・伝達失敗 | 出力契約（未解決事項・リスク欄の強制。埋まらない欄は「なし」と書く）（ルートの`AGENTS.md` 出力契約） | 防御あり |
| 他エージェントの成果の無視 | 状態外部化 + 引き継ぎテンプレ（既存作業の途中成果を破棄せず続きから）（`docs/templates.md`） | 防御あり |
| 曖昧な指示の増幅 | レビュー指摘を「実行可能な指示（ファイル+箇所+変更内容）」形式に強制（[../skills/recursive-review/SKILL.md](../skills/recursive-review/SKILL.md)） | 防御あり |
| 検証なしの合意（安易な同意） | レビュアー独立 + 「approve=無人反映の責任水準」の明示（ルートの`AGENTS.md` レビュー独立） | 防御あり |

**含意**: AI間の伝達を自由会話ではなく**構造化された成果物（verdict・テンプレ・指示形式）**に限定する設計が、
このカテゴリの主要因（自由会話での劣化）を構造的に回避している。

## 3. タスク検証の失敗（21.3%）

| MAST失敗様式（代表） | このハーネスでの防御手段（該当ルールへのリンク） | 状態 |
|---|---|---|
| 不完全な検証 | 外部信号必須 + 受け入れ条件を1項目ずつ照合（[principles.md](principles.md) §4 / [../skills/recursive-review/SKILL.md](../skills/recursive-review/SKILL.md)） | 防御あり |
| 誤った検証（甘い自己採点） | Maker-Checker分離（実装者≠レビュアー）（ルートの`AGENTS.md` レビュー独立） | 防御あり |
| 早すぎる終了 | 指名した検証を全通過することを終了条件にする（無関係な常時greenによる空虚な真を排除）（[principles.md](principles.md) §2） | 防御あり |

## 照合から出た結論（3点）

1. **設計品質は実測分類に照らしても高い** — 14様式の大半に構造的防御を持てる設計になっている。
2. **最も系統的な穴は「ステップ反復・無進展」（単独最大の 15.7%）** — ループ台帳が独自に到達する
   「無進展検出が最大のギャップ」という結論と、Berkeley の実測データが**独立に一致する**。
   [principles.md](principles.md) の4ハードガードで無進展検出を正本化しているのは、この一致に根拠がある。
3. **多くの失敗は組織設計に起因し、モデル改良では解決しない**（MAST の結論）。
   モデルの世代交代を待つのではなく、ループとハーネスを設計するアプローチは方向として正しい。
