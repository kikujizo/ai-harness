# mino-changeability-review — 変更容易性の補助走査レンズ

明示指定されたとき、コードの「安全に変更し続けられるか」を補助レンズとして走査する。機能バグの正誤やmerge可否を単独判定しない。

**コア原則**: 全文複製禁止。規範的正本は [docs/mino-skills/core/mino-core-principles.md](../../../docs/mino-skills/core/mino-core-principles.md) を参照する。

## ハーネス内での位置づけ

- このSkillは変更容易性に関する**補助走査レンズ**である。
- PRの最終レビュー判定は `.agents/skills/recursive-review/SKILL.md` の契約が行う。
- このSkillの所見は、recursive-reviewの「基準外の気づき」または補助所見として渡す。
- 全所見には、recursive-review側で今回修正 / 後回し / wontfix のディスポジションを付ける。
- このSkill単独ではmerge可否を判断せず、`REVIEW_VERDICT` を出さない。
- `recursive-review/SKILL.md` 本体は変更しない。

## 使いどき / 使わないとき

**使う**（明示指定時）: PRレビューで変更容易性・DDD寄り設計の深掘りが必要なとき／AI生成コードの設計負債を補助的に見たいとき。

**使わない**: 通常の最終PRレビューそのもの／機能バグ・仕様充足だけの確認／新規実装／merge可否判定。

## インプット

必要なもの: レビュー対象のdiff、ファイル、PR、または該当コード。可能なら、そのコードの目的と関連Issue。

足りないとき: 目的が読めないこと自体を所見にし、勝手に目的を確定しない。

## 手順

### STEP 1: 目的の推論

- このコードの目的を先に1〜3行で推論する。
- 根拠（命名、呼び出し元、Issue、コメント）を明記する。
- 目的が読めない場合は最重要所見にする。

### STEP 2: 6観点走査

| 観点 | 見るもの |
|---|---|
| カプセル化 | 生データ公開、setter過多、Tell Don't Ask違反 |
| 契約 | 不正状態で生成できる、null垂れ流し、範囲・単位が型で表現されない |
| 目的の混在 | 1クラス・1関数に複数目的、typeコード分岐 |
| 集約整合 | 集約ルートを迂回した内部変更、整合性の分散 |
| 命名 | Manager/Util/Common/Dataなど目的が伝わらない名前 |
| 重複と発散 | 同一ビジネスルールの複数箇所実装 |

観点ごとに「該当あり / なし / 確認不能」を書く。

### STEP 3: 優先度付け

- 各所見に影響度（高/中/低）と修正コスト（高/中/低）を付ける。
- 高影響×低コストを先頭に並べる。
- 同一根本原因の所見はまとめる。

### STEP 4: 改善案と契約テスト候補

- 上位所見にはBefore/Afterの改善スケッチを付ける。
- 将来守るべき契約テスト候補を自然言語で書く。
- 修正実行はしない。修正は承認済みIssueと担当指定後の別作業である。

## 反証ラウンド

1. 各指摘は具体的な行・シンボル・diffに遡れるか。
2. 趣味的な書き換えではなく変更容易性への影響を説明できるか。
3. 目的推論が間違っている可能性を検討したか。
4. フレームワーク規約やライブラリ制約を設計負債と誤認していないか。

該当した指摘は削除または修正し、維持する場合も理由を残す。

## 出力契約（中間成果物）

以下はこのSkillの中間成果物形式である。最終レビュー報告は `AGENTS.md` および `recursive-review` の出力契約に従う。

1. **コードの目的推論**（根拠付き）。
2. **所見表**: `| 箇所 | 観点 | 問題 | 影響度 | 修正コスト | 優先度 |`。
3. **上位所見のBefore/After改善スケッチ**。
4. **追加すべき契約テスト候補**。
5. **判断根拠・確認事項・保留事項**: 目的推論の仮定、反証ラウンド、削除・維持した指摘。

## 品質基準（Doneの定義）

- 6観点すべてに該当あり / なし / 確認不能がある。
- すべての所見が具体的な箇所に遡れる。
- 単独のmerge可否判定や `REVIEW_VERDICT` を出していない。
- 最終レビューへ渡す場合、recursive-review側でディスポジションできる粒度になっている。
